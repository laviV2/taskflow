🚀 TASKFLOW CLOUD DEPLOYMENT GUIDE
===================================

This project is configured for a professional deployment on Railway (Backend) and optional static hosting.

--- 1. BACKEND PREPARATION (Railway) ---
1. Create a New Project on Railway.app.
2. Connect your GitHub repository.
3. Set the following ENVIRONMENT VARIABLES:
   - DATABASE_URL = (Your MongoDB Connection String)
   - JWT_SECRET = (A long random string)
   - PORT = 8080 (Railway handles this automatically)

--- 2. FRONTEND PREPARATION ---
1. Go to the 'frontend' folder.
2. Run: npm run build
3. Copy all files from 'frontend/dist' to 'backend/public'.
   (This allows your backend to host the website itself!)

--- 3. DATABASE MIGRATION ---
Railway will automatically run 'npx prisma generate' during deployment.
To seed your initial data on the cloud:
1. Connect to your Railway terminal.
2. Run: npx prisma db seed

--- 4. PRODUCTION URL ---
Once deployed, update your Frontend's VITE_API_URL to point to your Railway domain:
Example: VITE_API_URL = https://taskflow-production.up.railway.app/api
